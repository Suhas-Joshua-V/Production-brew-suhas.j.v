📅 Day 1

IT Fundamentals & Systems

Building the foundation — understanding how computers, networks, and the internet actually work from the ground up.

📖

Overview

Today I have learned the basic IT terms and definitions that form the foundation of how technology works. The sir has explained how data flows in a complete cycle, starting from the client, moving to the server, and then interacting with the database before returning back to the user.

Overall, this day focused on building a strong base in understanding how systems and the internet operate at a fundamental level. It was theory-heavy, but the explanations were clear and helped me connect concepts I had heard before but never fully understood.

🧠

What I Learned

  • The basic IT terms and definitions that underpin how technology functions in the real world.
  • How the client-server model works — the client makes a request, the server processes it, and the database stores or retrieves data.
  • The role of the CPU (Central Processing Unit) — the brain of the computer that executes instructions and processes tasks.
  • The role of the GPU (Graphics Processing Unit) — originally for graphics, now also used for AI and parallel computing tasks.
  • How RAM (Random Access Memory) acts as short-term memory, holding data that the CPU is actively using.
  • The difference between HDD and SSD storage and why speed matters in modern systems.
  • How all these components interact together to run applications and process data smoothly.
  • A high-level overview of how the internet is structured and how data travels across networks.
"The sir has used simple analogies to explain complex ideas — for example, comparing RAM to a work desk and storage to a filing cabinet."
🛠️

What I Did

  • Attended the introduction session and got familiar with the workshop structure and schedule.
  • Took notes on IT terminology — hardware components, software layers, and networking basics.
  • Participated in a Q&A session where the instructor broke down the client-server cycle step by step.
  • Drew a simple diagram of how a web request flows: browser → server → database → response.
  • Discussed real-world examples of how these concepts apply to websites and apps we use daily.
