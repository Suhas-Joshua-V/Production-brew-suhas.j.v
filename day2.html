📅 Day 2

Websites, Servers & Terminal

From restaurant analogies to real SSH connections — today we explored how the web actually works and got our hands on real servers.

📖

Overview

Day 2 was a significant step up from the first day. We explored how websites work — not just the surface level, but the actual infrastructure behind them. The sir explained websites using a restaurant analogy, which genuinely made it click for me.

We also got to use the terminal for real, connected to an actual remote server using SSH, and worked inside the WordPress dashboard to create and publish pages. It was a busy day, but one of the most hands-on sessions of the workshop.

🧠

What I Learned

  • How a website is like a restaurant — the domain is the address, hosting is the kitchen, and the database is the pantry storing all the ingredients (data).
  • The difference between a webpage and a website — a webpage is a single document; a website is a collection of linked pages.
  • What static vs dynamic websites are — static sites serve the same content to everyone, dynamic sites generate content based on the user or data.
  • How domains and hosting work together — the domain is the name, hosting is the space where files live.
  • Networking concepts: IP addresses, DNS (translating domain names to IPs), and ports (specific channels for different services).
  • How to use terminal commands: pwd, ls, cd, and whoami to navigate a server's file system.
  • How SSH (Secure Shell) allows you to connect to and control a remote server securely from your local machine.
  • The role of Claude as an AI assistant — how it can help write, plan, and improve code and content.
"The restaurant analogy was genuinely one of the clearest explanations I've heard — it made every web concept suddenly feel logical and relatable."
🛠️

What I Did

  • Practised terminal commands in a live environment — navigated directories, listed files, and checked the current user.
  • Connected to a real remote server using SSH — this was a first for me and felt genuinely exciting.
  • Explored the file structure of the server after connecting, understanding where websites are typically hosted.
  • Opened and navigated the WordPress dashboard for the first time.
  • Created a new page inside WordPress, added content, and published it live.
  • Experimented with the WordPress editor and learned about posts vs pages, themes, and plugins at a surface level.
  • Had a brief introduction to Claude as an AI assistant and explored how it can be used during development.
